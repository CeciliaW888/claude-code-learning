Day 1: 架构总览 — 三层架构 · Tool 系统 · Agent 循环

🟢 入门 · 预计时间: 1小时

Claude Code 是一个基于终端的 AI 编程助手，理解其架构是深入学习的第一步。本节从宏观视角解析三层架构、Tool 系统和 Agent 循环的核心设计。

1

三层架构：表现层 / 核心层 / 服务层

表现层负责终端 UI（Ink/React），核心层包含 Agent 循环与 Tool 调度，服务层对接 Claude API 和文件系统等外部资源。三层各司其职，通过清晰的接口解耦。

2

Agent 循环：感知 → 思考 → 行动

Agent 循环是 Claude Code 的心脏。每一轮对话都经历：接收用户输入（感知）、调用 Claude API 推理（思考）、执行 Tool 操作（行动），然后将结果反馈给模型继续循环。

3

30+ Tool 统一接口

所有工具（BashTool、FileRead、FileWrite 等）都实现统一的 ToolDef 接口，包括名称、描述、参数 schema 和执行函数。这使得新增工具只需实现一个接口。

关键洞察： Claude Code 的架构本质上是一个「Tool-Augmented LLM Agent」——模型决定调用哪个工具，工具执行后将结果返回模型，形成闭环。

💻 核心代码

// Tool 统一接口定义
type ToolDef = {
  name: string;              // 工具名称，如 "Bash"
  description: string;       // 给 Claude 模型看的描述
  inputSchema: JSONSchema;   // 参数定义
  execute: (input: unknown) => Promise<ToolResult>;
};
