Day 8: BashTool 深度解析 — AST 安全解析 · 权限 · 沙盒

🟡 中级 · 预计时间: 1小时

BashTool 是 Claude Code 最强大也最危险的工具——它能执行任意 shell 命令。本节解析 BashTool 如何通过 AST 解析和多层安全机制在强大与安全之间取得平衡。

1

AST 解析检测危险命令

BashTool 不是用正则表达式匹配危险命令，而是将 shell 命令解析为 AST（抽象语法树），精确识别 rm -rf、curl | sh 等危险模式，避免正则匹配的误报和漏报。

2

多层安全防御

安全不是单一防线：① AST 解析检测危险模式 → ② 只读命令白名单 → ③ 权限系统询问用户 → ④ 沙盒环境隔离。任何一层拦截都能阻止危险操作。

3

输出截断策略

命令输出可能极长（如 find /），BashTool 会截断过长输出并保留头尾，告知模型输出被截断，避免撑爆上下文窗口。

关键洞察： 安全性不能依赖单一机制——AST 解析可能遗漏新模式，权限询问可能被用户误批准，只有多层防御才能构建可靠的安全体系。

💻 核心代码

// BashTool 安全检查流程简化版
async function executeBash(command: string): Promise<ToolResult> {
  // 1. AST 解析，检测危险模式
  const ast = parseShellScript(command);
  const risks = detectDangerousPatterns(ast);

  // 2. 只读命令白名单（ls, cat, grep 等直接放行）
  if (isReadOnlyCommand(ast)) {
    return executeInSandbox(command);
  }

  // 3. 有风险 → 请求用户授权
  if (risks.length > 0) {
    const approved = await requestPermission("Bash", command, risks);
    if (!approved) return { error: "用户拒绝执行" };
  }

  // 4. 在沙盒中执行
  return executeInSandbox(command);
}
