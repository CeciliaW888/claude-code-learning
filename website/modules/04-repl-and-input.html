Day 4: REPL 与输入 — 交互循环 · 斜杠命令 · Early Input

🟡 中级 · 预计时间: 1小时

REPL（Read-Eval-Print Loop）是 Claude Code 交互的核心引擎。本节深入解析用户输入如何被捕获、解析和分发，以及 Early Input 如何解决启动延迟问题。

1

REPL.tsx：核心交互引擎

REPL.tsx 是整个交互循环的入口。它管理输入框状态、处理用户提交、调度 Agent 循环，并在 Agent 返回结果后等待下一轮输入。

2

斜杠命令系统

用户输入以 / 开头时，会被路由到斜杠命令系统。内置命令如 /help、/clear、/compact 等，支持自定义扩展。命令和普通对话走不同的处理路径。

3

Early Input 预捕获

用户经常在 Claude Code 启动完成前就开始打字。Early Input 机制在启动阶段就开始监听键盘输入并缓存，初始化完成后立即使用缓存的输入，实现无缝体验。

关键洞察： Early Input 是一个优秀的 UX 设计模式——不要让用户等你准备好，而是在你准备的同时就开始接收用户输入。

💻 核心代码

// handleSubmit 简化版
async function handleSubmit(input: string) {
  // 斜杠命令走单独路径
  if (input.startsWith("/")) {
    return handleSlashCommand(input);
  }

  // 普通对话：添加用户消息，启动 Agent 循环
  addMessage({ role: "user", content: input });
  await runAgentLoop();
}
