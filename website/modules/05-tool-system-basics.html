Day 5: Tool 系统基础

🟡中级 · 预计时间: 1小时

Tool 系统是 Claude Code 能力的基石。每个工具都通过统一的 ToolDef 接口定义，让 Claude 知道如何调用它们。本节深入解析工具定义、构建和注册机制。

1
ToolDef 接口

每个工具必须定义 name、description、inputSchema 和 call 函数。这些字段构成了工具的完整描述，供 Claude 理解和调用。

2
description 是给 Claude 看的

description 字段不是给用户看的注释——它是 Claude 决定何时使用该工具的关键依据。写好 description 直接影响工具被正确调用的概率。

3
inputSchema 参数校验

使用 JSON Schema 定义工具的输入参数，包括类型、必填项和描述。Claude 会根据 Schema 生成正确的参数格式。

关键洞察： 工具的 description 本质上是写给 AI 的 prompt——它决定了 Claude 在什么场景下选择使用这个工具。

💻 核心代码

// buildTool 工具构建函数
function buildTool(config: ToolConfig): ToolDef {
  return {
    name: config.name,
    description: config.description,  // 给 Claude 的提示
    inputSchema: {
      type: "object",
      properties: config.params,
      required: config.required ?? [],
    },
    call: config.handler,
  };
}
