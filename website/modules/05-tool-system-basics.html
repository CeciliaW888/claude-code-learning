Day 5: Tool 系统基础 — ToolDef · buildTool · 生命周期

🟡 中级 · 预计时间: 1小时

Tool 系统是 Claude Code 的核心能力引擎——模型通过调用工具来读写文件、执行命令、搜索代码。本节深入解析工具的定义、构建和生命周期。

1

ToolDef：统一接口

每个工具都遵循 ToolDef 接口定义：name 标识工具，description 告诉 Claude 模型何时使用此工具，inputSchema 定义参数结构，execute 是实际执行逻辑。

2

description 是给 Claude 看的

工具的 description 不是给人看的文档，而是 prompt 的一部分——它告诉 Claude 模型什么场景该用这个工具、有什么限制、期望什么输入。写好 description 直接影响模型的工具选择准确度。

3

inputSchema 定义参数

使用 JSON Schema 格式定义工具的输入参数，包括类型、必填项、枚举值等。Claude 模型根据 schema 生成合法的工具调用参数。

关键洞察： Tool 的 description 本质上是 prompt engineering——好的描述让模型精准选择工具，差的描述导致模型误用或忽略工具。

💻 核心代码

// buildTool 工厂函数简化版
function buildTool(config: ToolConfig): ToolDef {
  return {
    name: config.name,
    description: config.description,  // 给 Claude 模型的使用说明
    inputSchema: config.inputSchema,  // JSON Schema 参数定义
    async execute(input) {
      // 权限检查 → 参数验证 → 执行 → 返回结果
      await checkPermission(config.name, input);
      const validated = validateInput(config.inputSchema, input);
      return config.handler(validated);
    },
  };
}
