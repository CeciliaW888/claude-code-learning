Day 18: 状态管理

🟡中级 · 预计时间: 1小时

状态管理决定了 Claude Code 如何追踪和更新应用的所有运行时数据。从 React Context Provider 到不可变更新模式，再到会话恢复机制。

1
Context Provider 模式

使用 React Context 在组件树中共享状态，避免 prop drilling。全局状态通过 Provider 注入，任何组件都可以访问。

2
不可变更新

状态更新采用不可变模式：每次修改创建新对象而非直接修改原对象。这确保状态变更可追踪，避免意外的副作用。

3
会话恢复

Claude Code 支持会话恢复：关闭终端后重新打开，可以继续之前的对话。状态被序列化到磁盘，下次启动时自动加载。

关键洞察： 不可变更新看似多此一举，实则是复杂状态管理的基石——它让每次变更都可预测、可回溯。

💻 核心代码

// AppState 状态定义
interface AppState {
  messages: Message[];
  model: string;
  tools: ToolDef[];
  sessionId: string;
  tokenUsage: { input: number; output: number };
}

// 不可变更新
function addMessage(state: AppState, msg: Message): AppState {
  return {
    ...state,
    messages: [...state.messages, msg],
  };
}

// Context Provider
const AppContext = createContext<AppState>(initialState);

function AppProvider({ children }: Props) {
  const [state, setState] = useState(initialState);
  return (
    <AppContext.Provider value={state}>
      {children}
    </AppContext.Provider>
  );
}
