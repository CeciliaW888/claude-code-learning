Day 18: 状态管理 — AppState · Provider · 会话恢复

🟡 中级 · 预计时间: 1小时

Claude Code 使用 React 的状态管理模式来管理整个应用的状态。本节解析 AppState 的设计、Context Provider 的层级结构，以及会话如何持久化和恢复。

1

React Context Provider

AppState 通过 React Context API 在组件树中共享。顶层 Provider 包裹整个应用，任何子组件都可以通过 useAppState() Hook 访问和更新全局状态。

2

不可变更新

状态更新遵循 React 的不可变原则：不直接修改 state 对象，而是创建新对象。这确保 React 能正确检测变化并触发重新渲染。

3

会话持久化与恢复

对话历史和状态会被序列化保存到磁盘。用户可以通过 claude --resume 恢复之前的会话，包括完整的消息历史和上下文。

关键洞察： Claude Code 在终端中使用了 Web 开发的状态管理模式（React Context + 不可变更新），这得益于 Ink 框架让终端 UI 也能享受 React 生态。

💻 核心代码

// AppState 与 useAppState 简化版
interface AppState {
  messages: Message[];
  tools: ToolDef[];
  permissionMode: PermissionMode;
  model: string;
  totalCost: number;
  sessionId: string;
}

const AppStateContext = React.createContext<AppState>(null!);

function AppStateProvider({ children }: { children: React.ReactNode }) {
  const [state, setState] = useState<AppState>(initialState);

  // 不可变更新：添加消息
  const addMessage = (msg: Message) => {
    setState(prev => ({
      ...prev,
      messages: [...prev.messages, msg],
    }));
  };

  return (
    <AppStateContext.Provider value={{ ...state, addMessage }}>
      {children}
    </AppStateContext.Provider>
  );
}

// 任何组件都可以访问状态
function useAppState() {
  return useContext(AppStateContext);
}
