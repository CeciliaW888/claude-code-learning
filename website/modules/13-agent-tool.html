Day 13: 子 Agent 工具 — Agent 调用 Agent · 前台/后台

🔴 高级 · 预计时间: 1小时

当任务过于复杂，一个 Agent 不够用时，Claude Code 允许主 Agent 派生子 Agent 来并行处理子任务。本节解析子 Agent 的创建、通信和生命周期管理。

1

主 Agent 派生子 Agent

AgentTool 是一个特殊的工具——它的执行逻辑是启动一个新的 Agent 实例。子 Agent 拥有独立的消息列表和工具集，但共享同一个项目上下文。主 Agent 可以给子 Agent 分配具体的子任务。

2

2 分钟自动后台化

子 Agent 默认在前台运行（用户可以看到其输出）。如果执行超过 2 分钟，会自动转入后台，避免阻塞主 Agent 的交互。用户可以随时查看后台任务的状态。

3

独立消息列表

子 Agent 维护自己的消息列表，不会污染主 Agent 的对话历史。子 Agent 完成后，只将最终结果（而非全部对话）返回给主 Agent，保持主对话的上下文精简。

关键洞察： 子 Agent 的独立消息列表设计是关键——如果子 Agent 的所有中间对话都加入主 Agent 的上下文，会迅速撑爆上下文窗口。

💻 核心代码

// AgentTool 简化版
const AgentTool: ToolDef = {
  name: "Agent",
  description: "启动子 Agent 处理独立的子任务",
  inputSchema: {
    type: "object",
    properties: {
      task: { type: "string", description: "子任务描述" },
    },
    required: ["task"],
  },
  async execute({ task }) {
    const subAgent = new QueryEngine({
      messages: [{ role: "user", content: task }],
      tools: getSubAgentTools(),  // 子 Agent 可用的工具子集
    });

    // 设置 2 分钟超时自动后台化
    const timeout = setTimeout(() => subAgent.moveToBackground(), 2 * 60 * 1000);

    const result = await subAgent.run();
    clearTimeout(timeout);

    return { output: result.summary };  // 只返回摘要给主 Agent
  },
};
