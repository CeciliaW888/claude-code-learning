Day 13: 子 Agent

🔴高级 · 预计时间: 1小时

Claude Code 可以让一个 Agent 调用另一个 Agent——子 Agent 模式。这种递归调用实现了复杂任务的分解与并行处理。

1
Agent 调用 Agent

主 Agent 可以将子任务委托给子 Agent 处理。子 Agent 拥有独立的消息历史和工具集，执行完毕后将结果返回给主 Agent。

2
2 分钟后台化

子 Agent 默认在前台运行。如果执行时间超过 2 分钟，系统自动将其转为后台任务，主 Agent 可以继续处理其他工作。

3
独立消息历史

子 Agent 拥有自己的消息列表，与主 Agent 隔离。这避免了上下文污染，让子 Agent 专注于分配的子任务。

关键洞察： 子 Agent 的 2 分钟后台化机制平衡了用户体验和任务效率——短任务即时返回，长任务不阻塞主流程。

💻 核心代码

// AgentTool 子 Agent 实现
const AgentTool: ToolDef = {
  name: "agent",
  description: "启动子 Agent 处理独立子任务",
  inputSchema: {
    type: "object",
    properties: {
      task: { type: "string", description: "子任务描述" },
    },
    required: ["task"],
  },
  call: async ({ task }) => {
    const subMessages: Message[] = [
      { role: "user", content: task },
    ];

    // 子 Agent 独立循环
    const result = await query(subMessages, tools);
    return { content: result };
  },
};
