Day 11: 上下文管理 — System Prompt · CLAUDE.md · 记忆

🟡 中级 · 预计时间: 1小时

Claude Code 的行为不仅由模型决定，更由精心构建的上下文塑造。本节解析 System Prompt 的动态组装、CLAUDE.md 的多级加载，以及上下文如何影响模型行为。

1

System Prompt 动态组装

System Prompt 不是一个静态字符串，而是动态拼接而成：基础指令 + 工具描述 + CLAUDE.md 内容 + 当前环境信息（OS、shell、cwd 等）。每次对话都根据当前状态组装。

2

CLAUDE.md 多级加载

CLAUDE.md 从多个位置加载并按优先级合并：用户级（~/.claude/CLAUDE.md）→ 父目录级 → 项目级（./CLAUDE.md）。项目级优先级最高，可以覆盖用户级的通用设置。

3

上下文收集

除了 CLAUDE.md，上下文还包括：git 状态、当前分支、最近的 commit、项目文件结构等。这些信息帮助模型理解项目背景，做出更准确的决策。

关键洞察： CLAUDE.md 是用户与模型之间的「契约」——你可以通过它告诉 Claude 项目的技术栈、编码规范、禁止事项等，让模型行为与团队要求一致。

💻 核心代码

// buildSystemPrompt 简化版
function buildSystemPrompt(state: AppState): string {
  const parts: string[] = [];

  // 基础指令
  parts.push(BASE_SYSTEM_PROMPT);

  // 工具描述（告诉模型可以用哪些工具）
  parts.push(formatToolDescriptions(state.tools));

  // CLAUDE.md 多级加载（项目级优先）
  const claudeMd = loadClaudeMd([
    `${state.homedir}/.claude/CLAUDE.md`,    // 用户级
    `${state.cwd}/../CLAUDE.md`,              // 父目录级
    `${state.cwd}/CLAUDE.md`,                 // 项目级（最高优先）
  ]);
  parts.push(claudeMd);

  // 环境信息
  parts.push(`OS: ${state.os}, Shell: ${state.shell}, CWD: ${state.cwd}`);

  return parts.join("\n\n");
}
