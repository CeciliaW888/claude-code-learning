Day 11: 上下文管理

🟡中级 · 预计时间: 1小时

上下文管理决定了 Claude 在每次对话中「知道什么」。System Prompt 的动态组装、CLAUDE.md 多级配置和记忆机制共同构成了上下文的完整图景。

1
System Prompt 动态组装

System Prompt 不是静态文本——它根据当前环境、可用工具、项目配置动态拼接而成，每次对话的 System Prompt 都可能不同。

2
CLAUDE.md 多级配置

CLAUDE.md 支持三级配置：用户级（~/.claude/CLAUDE.md）、项目级（项目根目录）、目录级（子目录）。项目级优先级最高。

3
记忆机制

Claude Code 通过 CLAUDE.md 实现持久记忆：将重要的项目规范、编码习惯写入文件，后续对话自动加载，无需重复说明。

关键洞察： CLAUDE.md 本质上是写给 AI 的「项目说明书」——它直接影响 Claude 的行为模式和输出质量。

💻 核心代码

// buildSystemPrompt 动态组装
function buildSystemPrompt(context: Context): string {
  const parts: string[] = [
    BASE_SYSTEM_PROMPT,
    buildToolDescriptions(context.tools),
    loadClaudeMd(context.projectDir),  // 项目级 CLAUDE.md
    loadClaudeMd(context.homeDir),     // 用户级 CLAUDE.md
    buildEnvironmentInfo(context),
  ];

  return parts.filter(Boolean).join("\n\n");
}
