Day 20: 多 Agent

🔴高级 · 预计时间: 1小时

当任务规模超出单个 Agent 的能力时，多 Agent 协作登场。Claude Code 支持 Team 模式和 Swarm 模式两种协作架构，并通过 Git Worktree 实现文件隔离。

1
Team 模式：Coordinator 协调

Team 模式有一个 Coordinator（协调者）Agent 负责分配任务、汇总结果。Worker Agent 各司其职，最终由 Coordinator 整合输出。

2
Swarm 模式：对等协作

Swarm 模式中所有 Agent 地位平等，通过共享消息总线通信。每个 Agent 自主决定接手哪些子任务，适合探索性工作。

3
Git Worktree 隔离

多个 Agent 同时修改代码会产生冲突。Git Worktree 为每个 Agent 创建独立的工作目录，让它们在同一仓库的不同分支上并行工作。

关键洞察： Git Worktree 是多 Agent 并行编码的关键——没有文件隔离，多个 Agent 修改同一文件必然产生冲突。

💻 核心代码

// Coordinator 协调者模式
class Coordinator {
  private workers: Agent[] = [];

  async dispatch(task: string): Promise<string> {
    // 分解任务
    const subtasks = await this.plan(task);

    // 为每个子任务创建 Worktree
    const results = await Promise.all(
      subtasks.map(async (sub) => {
        const worktree = await createWorktree(sub.branch);
        const worker = new Agent({ cwd: worktree });
        return worker.execute(sub.description);
      })
    );

    // 汇总结果
    return this.summarize(results);
  }
}
