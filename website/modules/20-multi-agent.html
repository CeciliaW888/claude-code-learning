Day 20: 多 Agent 系统 — Team · Swarm · Coordinator · Worktree

🔴 高级 · 预计时间: 1小时

单个 Agent 的能力有上限——真正复杂的项目需要多个 Agent 协作。本节解析多 Agent 系统的两种模式（Team 和 Swarm），以及 Git Worktree 如何隔离并行操作。

1

Team 模式：Coordinator 分层协作

Team 模式有一个 Coordinator（协调者）负责任务分解和分配。Coordinator 将大任务拆分为子任务，分配给多个 Worker Agent，收集结果后整合。适合有明确层级的复杂项目。

2

Swarm 模式：对等 Agent 网络

Swarm 模式中所有 Agent 是对等的，没有中心协调者。Agent 之间通过共享状态（如文件系统）进行隐式协作。适合松耦合的并行任务。

3

Git Worktree 隔离并行操作

多个 Agent 同时修改代码会产生冲突。Git Worktree 为每个 Agent 创建独立的工作目录（共享同一个 .git 仓库），每个 Agent 在自己的分支上工作，最后合并。

关键洞察： Git Worktree 是多 Agent 并行编码的关键基础设施——它解决了「多个 Agent 同时编辑同一个文件」的根本冲突问题。

💻 核心代码

// Coordinator 模式简化版
async function coordinatorLoop(task: string, workerCount: number) {
  // 1. Coordinator 分解任务
  const subtasks = await decompose(task);

  // 2. 为每个 Worker 创建独立的 Git Worktree
  const worktrees = await Promise.all(
    subtasks.map((_, i) =>
      exec(`git worktree add ../worker-${i} -b worker-branch-${i}`)
    )
  );

  // 3. 并行启动 Worker Agent
  const results = await Promise.all(
    subtasks.map((subtask, i) => {
      const worker = new AgentWorker({
        cwd: worktrees[i].path,  // 各自独立的工作目录
        task: subtask,
      });
      return worker.run();
    })
  );

  // 4. 合并所有 Worker 的结果
  for (const worktree of worktrees) {
    await exec(`git merge ${worktree.branch}`);
    await exec(`git worktree remove ${worktree.path}`);
  }
}
