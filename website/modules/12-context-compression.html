Day 12: 上下文压缩

🔴高级 · 预计时间: 1小时

当对话变长，token 数逼近上下文窗口限制时，压缩机制自动启动。Claude Code 设计了三级压缩策略，在保留关键信息的同时控制成本。

1
三级压缩：Snip / MicroCompact / FullCompact

Snip 移除工具输出的冗余部分；MicroCompact 压缩早期对话；FullCompact 对整个对话历史进行摘要重写。三级递进，逐步加大压缩力度。

2
80% 阈值触发

当 token 使用量达到上下文窗口的 80% 时，自动触发压缩。留出 20% 的余量，确保后续对话有足够空间。

3
Token 预算管理

系统持续追踪 token 消耗，为 System Prompt、工具描述、对话历史分配预算，确保每个部分都在合理范围内。

关键洞察： 压缩不是简单截断——它是智能摘要，保留关键决策和上下文，丢弃冗余的中间过程。

💻 核心代码

// 上下文压缩逻辑
function shouldCompress(messages: Message[], limit: number): boolean {
  const totalTokens = countTokens(messages);
  return totalTokens > limit * 0.8;  // 80% 阈值
}

async function compress(messages: Message[]): Promise<Message[]> {
  // 第一级：Snip 冗余工具输出
  let result = snipToolOutputs(messages);

  if (stillOverLimit(result)) {
    // 第二级：MicroCompact 早期对话
    result = microCompact(result);
  }

  if (stillOverLimit(result)) {
    // 第三级：FullCompact 全面摘要
    result = await fullCompact(result);
  }

  return result;
}
