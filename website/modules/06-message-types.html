Day 6: 消息类型

🟡中级 · 预计时间: 1小时

Claude Code 中的每条消息都有特定类型和结构。理解消息类型是掌握 Agent 通信机制的关键，特别是工具调用与结果的配对机制。

1
5 种消息类型

系统中有 5 种消息：system（系统提示）、user（用户输入）、assistant（AI 回复）、tool_use（工具调用）、tool_result（工具结果）。

2
tool_use / tool_result 配对

每次工具调用会生成一个 tool_use 消息，执行完成后返回对应的 tool_result。两者通过唯一的 ID 严格配对。

3
ID 匹配机制

tool_use 生成唯一的 tool_use_id，tool_result 必须携带相同的 ID。API 会拒绝 ID 不匹配的消息对，确保消息链完整性。

关键洞察： tool_use_id 是工具调用系统的关键纽带——没有正确的 ID 匹配，整个 Agent 循环就会中断。

💻 核心代码

// 消息类型定义
type Message =
  | { role: "system"; content: string }
  | { role: "user"; content: string }
  | { role: "assistant"; content: ContentBlock[] }

type ContentBlock =
  | { type: "text"; text: string }
  | { type: "tool_use"; id: string; name: string; input: unknown }
  | { type: "tool_result"; tool_use_id: string; content: string }
