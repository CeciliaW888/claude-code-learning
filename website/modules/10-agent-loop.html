Day 10: Agent 循环

🔴高级 · 预计时间: 1小时

Agent 循环是 Claude Code 的心脏。它决定了 AI 如何持续思考、调用工具并最终给出答案。理解 query() 函数和流式响应机制，是掌握 Agent 系统的核心。

1
QueryEngine 引擎

QueryEngine 封装了与 Claude API 的交互逻辑，管理消息历史、工具定义和模型配置，是 Agent 循环的控制中枢。

2
while(true) 循环

Agent 循环的核心是一个 while(true)：调用 API → 检查响应 → 如果有工具调用则执行并继续，如果是 end_turn 则退出循环。

3
流式响应

使用 Server-Sent Events 实现流式输出，让用户实时看到 AI 的思考过程，而非等待完整响应。提升感知速度和交互体验。

关键洞察： Agent 不是单次问答——它是一个 while(true) 循环，不断调用工具直到任务完成，这正是"Agent"与"Chatbot"的本质区别。

💻 核心代码

// query() 核心循环
async function query(messages: Message[], tools: ToolDef[]) {
  while (true) {
    const response = await client.messages.create({
      model: "claude-sonnet-4-20250514",
      messages,
      tools: tools.map(toAPIDef),
      stream: true,
    });

    // 处理流式响应
    for await (const event of response) {
      handleStreamEvent(event);
    }

    if (response.stop_reason === "end_turn") break;

    // 执行工具调用，结果加入消息历史
    await executeToolCalls(response, messages);
  }
}
