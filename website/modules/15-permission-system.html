Day 15: 权限系统 — 多层安全 · 权限模式 · 沙盒

🔴 高级 · 预计时间: 1小时

权限系统是 Claude Code 安全架构的核心。本节解析四种权限模式的设计理念、多层防御机制，以及 CLAUDE.md 中的权限规则如何与代码层面的安全检查协同工作。

1

四种权限模式

default：危险操作逐次询问用户；plan：只允许读取操作，所有写入操作需确认（最安全）；auto：自动批准大多数操作；bypass：完全跳过权限检查（仅开发环境）。

2

多层安全防御

安全检查从外到内层层递进：① CLAUDE.md 规则（禁止特定操作）→ ② 权限模式（决定是否询问）→ ③ 工具级检查（AST 解析等）→ ④ 沙盒隔离（限制系统访问）。

3

CLAUDE.md 权限规则

用户可以在 CLAUDE.md 中定义权限规则，如「永远不要删除文件」「只在 src/ 目录下编辑」等。这些规则在权限检查时被读取并强制执行。

关键洞察： plan 模式是代码审查的最佳搭档——Claude 只能读取和分析代码，所有修改操作都需要人工确认，确保每一个变更都经过人类审批。

💻 核心代码

// 权限检查流程简化版
async function checkPermission(
  toolName: string,
  input: unknown,
  mode: PermissionMode
): Promise<boolean> {
  // 1. CLAUDE.md 规则检查
  const rules = loadPermissionRules();
  if (rules.isDenied(toolName, input)) {
    return false;  // 被 CLAUDE.md 规则明确禁止
  }

  // 2. 权限模式检查
  switch (mode) {
    case "bypass": return true;           // 跳过所有检查
    case "auto":   return true;           // 自动批准
    case "plan":
      return isReadOnlyTool(toolName);    // 只允许读取
    case "default":
      if (isReadOnlyTool(toolName)) return true;
      return await promptUser(            // 询问用户
        `允许 ${toolName} 执行此操作？`
      );
  }
}
