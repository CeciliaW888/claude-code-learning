Day 15: 权限系统

🔴高级 · 预计时间: 1小时

权限系统是 Claude Code 安全架构的核心。它通过多层防御机制确保 AI 不会执行未经授权的操作，同时在安全与效率之间找到平衡。

1
4 种权限模式

Claude Code 提供四种运行模式：plan（只读规划）、ask（需确认）、auto（自动执行）和 custom（自定义规则）。安全性从高到低递减。

2
多层防御

权限检查贯穿多个层级：工具级别（工具本身的权限声明）、会话级别（当前模式配置）、用户级别（全局安全规则）。任意一层拒绝即阻止操作。

3
CLAUDE.md 权限规则

通过 CLAUDE.md 可以定义项目级权限规则，如禁止删除特定目录、限制网络访问等，让安全策略与项目需求匹配。

关键洞察： 好的权限系统不是一味限制——而是让安全操作尽量自动化，仅对高风险操作要求人工确认。

💻 核心代码

// 权限检查逻辑
type PermissionMode = "plan" | "ask" | "auto" | "custom";

async function checkPermission(
  tool: ToolDef,
  input: unknown,
  mode: PermissionMode
): Promise<boolean> {
  // plan 模式：只允许只读操作
  if (mode === "plan" && tool.sideEffects) {
    return false;
  }

  // ask 模式：需要用户确认
  if (mode === "ask") {
    return await promptUser(`允许执行 ${tool.name}?`);
  }

  // auto 模式：自动放行（仅限已信任工具）
  if (mode === "auto" && tool.trusted) {
    return true;
  }

  return await promptUser(`确认执行?`);
}
