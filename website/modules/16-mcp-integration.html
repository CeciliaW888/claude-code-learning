Day 16: MCP 协议集成 — Model Context Protocol · 工具发现

🔴 高级 · 预计时间: 1小时

MCP（Model Context Protocol）是 Anthropic 提出的开放协议，让 AI 应用以标准化方式连接外部工具和数据源。本节解析 Claude Code 如何集成 MCP，实现动态工具扩展。

1

MCP 三种能力：Tools / Resources / Prompts

Tools：可执行的操作（如查询数据库、发送邮件）；Resources：可读取的数据源（如文件、API 响应）；Prompts：预定义的 prompt 模板。三种能力覆盖了 AI 与外部世界交互的主要场景。

2

Server / Client 架构

MCP Server 提供能力（工具/资源/提示），MCP Client（Claude Code）发现并调用这些能力。通信使用 JSON-RPC 协议，支持 stdio 和 HTTP 两种传输方式。

3

动态工具注册

Claude Code 启动时连接配置的 MCP Server，获取其提供的工具列表，并将这些工具动态注册到 Tool 系统中。模型看到的工具列表会包含 MCP 提供的外部工具。

关键洞察： MCP 让 Claude Code 的能力变成「可插拔」的——你不需要修改 Claude Code 源码，只需编写 MCP Server 就能让它连接新的数据源和工具。

💻 核心代码

// MCP Server 示例：提供一个查询数据库的工具
import { McpServer } from "@modelcontextprotocol/sdk/server/mcp.js";
import { StdioServerTransport } from "@modelcontextprotocol/sdk/server/stdio.js";
import { z } from "zod";

const server = new McpServer({
  name: "database-tools",
  version: "1.0.0",
});

// 注册一个工具
server.tool(
  "query_database",
  "执行 SQL 查询并返回结果",
  { sql: z.string().describe("要执行的 SQL 查询语句") },
  async ({ sql }) => {
    const result = await db.query(sql);
    return { content: [{ type: "text", text: JSON.stringify(result) }] };
  }
);

// 启动 Server
const transport = new StdioServerTransport();
await server.connect(transport);
