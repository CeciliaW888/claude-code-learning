Day 9: 文件工具

🟡中级 · 预计时间: 1小时

文件操作是编程助手的核心能力。Claude Code 提供三种文件工具——Read、Write、Edit——覆盖所有文件操作场景，其中 Edit 的设计尤为精巧。

1
Read / Write / Edit 三种工具

Read 读取文件内容，Write 创建或覆写文件，Edit 精确修改文件局部内容。三者分工明确，适用不同场景。

2
old_str / new_str 编辑模式

Edit 工具使用文本匹配而非行号定位：指定 old_str（要替换的文本）和 new_str（替换后的文本），精确定位修改位置。

3
行号辅助定位

Read 工具返回带行号的内容，帮助 Claude 精确定位代码位置。但编辑时用文本匹配而非行号，避免行号偏移导致的错误。

关键洞察： 用文本匹配代替行号编辑，是因为多次编辑会导致行号偏移——文本匹配天然避免了这个问题。

💻 核心代码

// FileEdit 工具核心逻辑
const FileEditTool: ToolDef = {
  name: "file_edit",
  description: "精确编辑文件中的指定文本",
  inputSchema: {
    type: "object",
    properties: {
      file_path: { type: "string" },
      old_str: { type: "string" },   // 要替换的原文
      new_str: { type: "string" },   // 替换后的内容
    },
    required: ["file_path", "old_str", "new_str"],
  },
  call: async ({ file_path, old_str, new_str }) => {
    const content = await readFile(file_path);
    const updated = content.replace(old_str, new_str);
    await writeFile(file_path, updated);
    return { success: true };
  },
};
