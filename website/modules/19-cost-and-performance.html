Day 19: 成本与性能

🟡中级 · 预计时间: 1小时

使用 AI 编程助手意味着真金白银的 API 费用。理解成本追踪、启动优化和 Prompt Cache 机制，帮助你在效果与成本之间找到最佳平衡。

1
成本追踪

系统实时追踪每次 API 调用的 token 消耗和费用。用户可以随时查看当前会话的累计成本，避免意外超支。

2
启动优化

通过动态 import、并行预加载和延迟初始化，Claude Code 将启动时间控制在秒级以内。只加载当前需要的模块。

3
Prompt Cache

Anthropic API 支持 Prompt Cache：相同的前缀内容（如 System Prompt）可以被缓存，后续请求直接复用，减少 90% 的前缀 token 费用。

关键洞察： Prompt Cache 是最有效的成本优化手段——只要 System Prompt 保持稳定，后续每次请求都能大幅节省费用。

💻 核心代码

// 成本追踪实现
interface CostTracker {
  sessionCost: number;
  totalInputTokens: number;
  totalOutputTokens: number;
}

function updateCost(tracker: CostTracker, usage: TokenUsage) {
  const inputCost = usage.input_tokens * PRICE_PER_INPUT_TOKEN;
  const outputCost = usage.output_tokens * PRICE_PER_OUTPUT_TOKEN;

  // Prompt Cache 折扣
  const cacheSavings = usage.cache_read_tokens
    ? usage.cache_read_tokens * PRICE_PER_INPUT_TOKEN * 0.9
    : 0;

  tracker.sessionCost += inputCost + outputCost - cacheSavings;
  tracker.totalInputTokens += usage.input_tokens;
  tracker.totalOutputTokens += usage.output_tokens;
}
