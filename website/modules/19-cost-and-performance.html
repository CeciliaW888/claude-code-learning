Day 19: 成本与性能 — 成本追踪 · 启动优化 · Token 预算

🟡 中级 · 预计时间: 1小时

AI Agent 的运行成本不可忽视——每次 API 调用都消耗 token 费用。本节解析 Claude Code 如何追踪成本、优化启动性能，以及利用 Prompt Cache 降低开销。

1

实时成本追踪

每次 API 调用后，系统记录输入/输出 token 数量并按模型定价计算费用。累计费用实时显示在终端状态栏中，让用户对当前会话的花费一目了然。

2

并行预加载启动优化

启动时使用 Promise.all() 并行加载配置、检查权限、初始化 API 客户端。配合动态 import()，首次交互延迟从数秒降至毫秒级。

3

Prompt Cache

Claude API 支持 Prompt Cache：System Prompt 和消息前缀在多次调用之间可以被缓存，避免重复处理相同的 token，降低延迟和费用（缓存 token 费用约为原价的 10%）。

关键洞察： Prompt Cache 是 Claude Code 成本优化的最大杠杆——System Prompt 和对话历史在每轮都重复发送，缓存后只需支付首次的 token 费用。

💻 核心代码

// 成本追踪实现简化版
interface CostTracker {
  inputTokens: number;
  outputTokens: number;
  cacheReadTokens: number;
  totalCost: number;
}

function updateCost(tracker: CostTracker, usage: ApiUsage): CostTracker {
  const inputCost = usage.input_tokens * PRICE_PER_INPUT_TOKEN;
  const outputCost = usage.output_tokens * PRICE_PER_OUTPUT_TOKEN;
  // 缓存 token 费用约为原价的 10%
  const cacheCost = usage.cache_read_tokens * PRICE_PER_INPUT_TOKEN * 0.1;

  return {
    inputTokens: tracker.inputTokens + usage.input_tokens,
    outputTokens: tracker.outputTokens + usage.output_tokens,
    cacheReadTokens: tracker.cacheReadTokens + usage.cache_read_tokens,
    totalCost: tracker.totalCost + inputCost + outputCost + cacheCost,
  };
}
