Day 2: 入口与启动

🟢入门 · 预计时间: 1小时

当你在终端输入 claude 命令时，背后发生了什么？本节将解析 CLI 入口、参数解析、并行预加载策略以及核心 State 对象的初始化过程。

1
Commander.js 参数解析

使用 Commander.js 库解析命令行参数，支持 --model、--verbose 等选项，将用户输入转换为结构化配置对象。

2
并行预加载

启动时并行执行多个初始化任务（加载配置、检查认证、读取 CLAUDE.md），而非串行等待，大幅缩短启动时间。

3
State 对象

所有运行时状态集中在一个 State 对象中管理，包括消息历史、当前模型、工具列表等，是整个应用的数据中心。

关键洞察： 并行预加载是 Claude Code 启动速度快的关键——多个独立任务同时执行，而不是排队等待。

💻 核心代码

// CLI 入口简化示意
import { Command } from "commander";

const program = new Command()
  .option("--model <model>", "指定模型")
  .option("--verbose", "详细输出")
  .parse(process.argv);

// 并行预加载
const [config, auth, claudeMd] = await Promise.all([
  loadConfig(),
  checkAuth(),
  readClaudeMd(),
]);
