Day 17: 任务系统

🔴高级 · 预计时间: 1小时

任务系统管理 Claude Code 中所有异步操作的生命周期。前台任务可中断，后台任务静默执行，多个任务可以并行运行。

1
前台任务：可中断

前台任务在用户视野内执行，实时显示进度。用户可以随时中断（Ctrl+C），系统会安全地停止执行并回滚未完成的操作。

2
后台任务：静默执行

后台任务在用户不可见的情况下运行，完成后通知用户。适合耗时操作，如大规模代码搜索或长时间运行的子 Agent。

3
并行执行

多个任务可以同时运行，互不干扰。系统追踪每个任务的状态（pending/running/completed/failed），确保资源合理分配。

关键洞察： 前台/后台任务的区分让用户既能实时监控重要操作，又不会被长时间任务阻塞。

💻 核心代码

// Task 接口定义
interface Task {
  id: string;
  type: "foreground" | "background";
  status: "pending" | "running" | "completed" | "failed";
  description: string;
  cancel: () => void;
  result?: unknown;
}

class TaskManager {
  private tasks = new Map<string, Task>();

  async run(task: Task): Promise<unknown> {
    this.tasks.set(task.id, task);
    task.status = "running";

    try {
      const result = await task.execute();
      task.status = "completed";
      return result;
    } catch (err) {
      task.status = "failed";
      throw err;
    }
  }
}
