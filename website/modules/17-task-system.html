Day 17: 任务系统 — 前台/后台 · 进度追踪 · 并行

🔴 高级 · 预计时间: 1小时

复杂的编程任务往往需要同时进行多个操作。Claude Code 的任务系统管理前台和后台任务的生命周期，支持并行执行和进度追踪。

1

前台任务：可见可中断

前台任务的输出实时显示在终端中，用户可以看到执行进度并随时中断（Ctrl+C）。典型前台任务包括文件编辑、代码生成等需要用户关注的操作。

2

后台任务：静默通知

后台任务在不打扰用户的情况下静默执行。完成后通过通知告知用户结果。长时间运行的工具调用、子 Agent 超时后的任务都会转入后台。

3

并行执行

当模型在一次响应中请求多个 tool_use 时，如果工具之间没有依赖关系，系统会使用 Promise.all() 并行执行，显著提升效率。

关键洞察： 前台/后台的切换不是全有或全无——一个任务可以从前台开始（让用户看到初始进展），超时后自动转后台（不阻塞交互）。

💻 核心代码

// Task 接口定义
interface Task {
  id: string;
  type: "foreground" | "background";
  status: "running" | "completed" | "failed" | "cancelled";
  progress?: number;        // 0-100 进度百分比
  result?: ToolResult;
  startedAt: number;
  completedAt?: number;
}

// 并行执行多个工具
async function executeToolsInParallel(
  toolUseBlocks: ToolUseBlock[]
): Promise<ToolResult[]> {
  return Promise.all(
    toolUseBlocks.map(block => {
      const tool = findTool(block.name);
      const task = createTask(block, "foreground");
      return tool.execute(block.input)
        .then(result => { task.status = "completed"; return result; })
        .catch(err => { task.status = "failed"; return { error: err.message }; });
    })
  );
}
